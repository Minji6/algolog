## 개념 이해 과정

처음에는 왜 `(K+1)`로 나누는지 감이 안 잡혔다.

가장 큰 수와 두 번째 큰 수가 더해지는 패턴을 보니,  
K=3일 때 `6 6 6 5` 라는 묶음이 반복된다는 걸 알게 됐다.

- 묶음의 길이 = `K + 1` (가장 큰 수 K번 + 두 번째 큰 수 1번)
- M을 (K+1)로 나눈 **몫** = 묶음 반복 횟수
- M을 (K+1)로 나눈 **나머지** = 마지막에 잘린 개수

잘린 자리가 왜 항상 가장 큰 수부터 시작하는지도 헷갈렸는데,  
묶음이 `큰 수 K개 → 두 번째 수 1개` 순서로 구성되어 있어서  
묶음이 완성되지 않으면 뒤의 두 번째 수는 등장하지 않기 때문이다.

---

## 어려웠던 부분 & 원인 파악

### 어려웠던 것
1. **몫/나머지 개념** - 나눗셈 결과를 수열 반복 횟수와 연결하는 게 바로 안 됐다
2. **잘린 자리가 왜 큰 수부터인지** - 묶음 구조를 머릿속으로 그리기 전까지 납득이 안 됐다


### 왜 어려웠나 (원인)
- **예시 없이 추상적으로만 생각했다** → `6 6 6 5` 처럼 수열을 직접 써보니까 바로 보였다
- **개념을 수식으로 연결하는 게 낯설었다** → "묶음 반복 횟수 = 몫" 이라는 연결고리를 못 찾고 있었다
- **나눗셈 몫/나머지를 코드랑 연결 못했다** → `int(m / (k+1))`가 몫이고 `m % (k+1)`가 나머지라는 걸 코드 레벨에서 따로 확인하고 나서야 연결됐다

### 다음에 비슷한 유형 만나면
- 반복 패턴이 보이면 **묶음을 직접 써보기**
- 묶음 길이 찾으면 → `M ÷ 묶음길이` 로 몫/나머지 바로 적용

## 틀린 부분 & 교정

### 검증 문제: [7, 4, 2], N=3, M=11, K=3

| 단계 | 내 답 | 정답 | 비고 |
|------|-------|------|------|
| 묶음 구성 | 7774 | 7774 | ✅ |
| 묶음 반복 횟수 | 2번 | 2번 | ✅ |
| 나머지 | 3 | 3 | ✅ |
| 7이 더해지는 횟수 | 6번 | 6번 | ✅ |
| 최종 result | **71** | **62** | ❌ |

**오답 원인:** 마지막 계산 실수  
- `7 × 6 = 42`, `4 × 5 = 20` → `42 + 20 = 62`

## 나의 초기 풀이 & 개선점

### 초기 코드 (반복문 방식)

```python
n, m, k = map(int, input().split())
arr = list(map(int, input().split()))
arr.sort(reverse=True)
first = arr[0]
second = arr[1]
result = 0
count = 0  # 가장 큰 수가 연속으로 더해진 횟수

for i in range(m):
    if count < k:
        result += first
        count += 1
    else:
        result += second
        count = 0

print(result)
```

### 문제점 1: 시간복잡도

| | 초기 풀이 | 책 풀이 |
|---|---|---|
| 방식 | M번 직접 반복 | 수식 계산 |
| 시간복잡도 | O(M) | O(1) |
| M=100,000,000 | 약 수십 초 | 즉시 |

M이 매우 큰 수일 경우 시간 초과가 날 수 있다.  
→ **반복 패턴을 수식으로 바꾸는 게 핵심**이었다.

### 문제점 2: count 비교 방식의 엣지케이스

로직 자체는 맞지만 `first == second` 인 경우 (예: 배열에 같은 숫자가 있을 때) 를 별도로 고려하지 않았다.  
이 문제에서는 결과에 영향이 없지만, 문제 조건을 꼼꼼히 확인하는 습관이 필요하다.

---

## 최종 코드

```python
n, m, k = map(int, input().split())
data = list(map(int, input().split()))

data.sort()
first = data[n - 1]   # 가장 큰 수
second = data[n - 2]  # 두 번째로 큰 수

# 가장 큰 수가 더해지는 횟수 계산
count = int(m / (k + 1)) * k
count += m % (k + 1)

result = 0
result += count * first          # 가장 큰 수 더하기
result += (m - count) * second   # 두 번째로 큰 수 더하기

print(result)
```
